# RhythmFallServer

Локальный Flask‑сервер для RhythmFall, который анализирует аудио и возвращает автоматически сгенерированные ноты. Запускается на том же устройстве, что и клиент Godot.

Языки: [English](./README.md) | Русский

Примечание: Этот репозиторий содержит серверную часть RhythmFall. Клиентская часть (Godot) доступна здесь: https://github.com/abletoburntheweb/RhythmFall

Модели: В репозиторий не входят. Скачайте архивы из Releases (https://github.com/abletoburntheweb/RhythmFallServer/releases) и распакуйте в каталог rhythmfall-server/models.

## Что это
- Небольшой локальный HTTP‑сервер (Flask) для анализа аудио и генерации нот.
- Предоставляет эндпоинты для определения BPM и генерации ударных паттернов.
- Работает на localhost; треки не отправляются во внешние сервисы.

## Как это работает
- Принимает аудиофайл через HTTP.
- Оценивает темп, при необходимости разделяет на стемы, детектирует ударные, применяет жанровые паттерны.
- Возвращает JSON с нотами и простую статистику клиенту.

## Быстрый старт
1) Создать виртуальное окружение
```bash
python -m venv .venv
```
2) Активировать окружение
- Windows (cmd):
  ```cmd
  .venv\Scripts\activate
  ```
- Windows (PowerShell):
  ```powershell
  .venv\Scripts\Activate.ps1
  ```
3) Установить зависимости
```bash
pip install -r requirements.txt
```
4) Запустить сервер
```bash
python run.py
```
Откройте http://127.0.0.1:5000 — должно отобразиться: {"message": "RhythmFallServer is running"}

## Эндпоинты (кратко)
- POST /analyze_bpm — принимает аудио, возвращает { "bpm": число }
- POST /generate_drums — принимает аудио и метаданные, возвращает JSON с нотами

## Примечания
- По умолчанию сервер слушает localhost:5000; остановка — Ctrl+C.
- Обработка полностью локальная — треки не уходят в сеть.

## Зависимости
- Базовые: Python 3.10+, Flask, librosa (см. requirements.txt).
- Опциональные (улучшают качество/тайминг):
  - demucs — разделение на стемы для более чистой детекции ударных (требует PyTorch; рекомендуется ffmpeg).
  - madmom — бит‑треккинг для стабильного выравнивания.
  - Essentia (TempoCNN) — устойчивое определение темпа и аудио‑признаки.
  - onnxruntime — запуск ONNX‑голов для жанровой классификации.
  - Эти пакеты уже перечислены в requirements.txt; установка может потребовать доп. системных библиотек (например, PyTorch для demucs, ffmpeg).

## Модели для классификации жанров (опционально)
- Поддерживается Discogs400 (голова + лейблы и MAEST‑эмбеддер), если модели доступны.
- Ожидаемые файлы в папке модели (по умолчанию: models/genre_discogs400-discogs-maest-10s-pw-1):
  - <модель>.pb или <модель>.onnx (классификатор),
  - <модель>.json (лейблы/метаданные),
  - граф MAEST .pb (может лежать в этой же папке или соседней “maest”).
- Переменные окружения:
  - RF_DISCOGS400_DIR — путь к папке модели,
  - RF_MAEST_EMBED_PB — путь к графу эмбеддера MAEST.

## Платформы
- Основное тестирование проводилось на Ubuntu; на Windows некоторые опциональные пакеты могут требовать дополнительной настройки.
- Сервер работает и без опциональных компонентов — они подключаются динамически, если обнаружены.
